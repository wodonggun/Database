# Database

- DB를 통해 데이터를 가공해서 다양한 일들을 할 수 있음.
- 대규모의 데이터로부터 통찰력있는 분석결과를 뽑아낼 수 있음.


과거에는 파일을 저장하고, Open하여 정보들을 저장하였지만, 한계점이 존재하여 데이터베이스 시스템이란거를 고안하게됨.
데이터베이스는 소중한 정보들을 안전하고, 편하게 사용할 수 있음.

mysql, Oracle, mariadb, mongoDB, 


- 데이터베이스는 매우 복잡하고, 중요하여 위험하게 간주하여 사용하여야 합니다.
- 데이터베이스 수업을 진행하면서 왜 이걸 알아야하며, 어떻게 이걸 활용할 수 있는지 생각하고 사소한거라도 질문하고 이해해야함. 
- 이해를 통해서 공감을 할 수 있어야함. 공감이 없는 이해는 모래위에 지어진 성이라는 것을 기억해야합니다.




# 데이터베이스 본질이란?

데이터베이스가 복잡하지만, 아무리 어렵고 복잡한 이론도 그 중심에 자리잡고 있는 핵심은 복잡하지 않습니다.
데이터베이스의 대부분의 기능을 Input과 Output입니다. 
C: Create 
R: Read 
U: Update 
D: Delete  

# File vs Database  

탐색기에 파일 5개(사람 프로필정보)를 생성하여  
만약에 1억개라면 하나하나 열어보고 찾는다면 시간이 오래걸림.  
  
엑셀을 통해서 특정 저자의 사람을 보고 싶다면 필터를 사용함.  
특정 컬럼만을 가공해서 보여주려면 컬럼 Hide를 통해서 가능함.  
  
여러사람들이 데이터를 보고싶으면 엑셀파일을 모두가 공유를 해야함. 
공유를 했다가 특정 사용자가 실수로 삭제한다면 데이터가 날라감.  

데이터 베이스는 사용자가 원하는 데이터를 필터하고, 원하는 컬럼만 읽어와서 표현할 수 있음.  
또한 사용자에 따라 권한을 분리하여 데이터를 가공할 수 있다.
